Core Concepts

Matching theory studies conflict-free pair selection in graphs.

🔗

Independent Edges

Edges are independent when no two selected edges are incident with the same vertex.

\[ e_i=uv,\;e_j=xy,\quad \{u,v\}\cap\{x,y\}=\varnothing \]
🤝

Matching

A matching is a set of independent edges. It represents conflict-free pairing.

\[ M=\{e_1,e_2,\ldots,e_k\} \]
✅

Saturated Vertex

A vertex is saturated by a matching if it is incident with an edge of the matching.

\[ v\text{ saturated}\Longleftrightarrow \exists e\in M,\;v\in e \]
📌

Maximal Matching

A maximal matching cannot be enlarged by adding another edge while preserving the matching property.

\[ M\cup\{e\}\text{ is not a matching for all }e\in E(G)-M \]
🏆

Maximum Matching

A maximum matching has the largest number of edges among all matchings of the graph.

\[ |M|=\alpha'(G) \]
🛡️

Covering

A covering is a selected set that touches or covers the required graph elements through incidence.

\[ \beta(G),\;\beta'(G)\quad\text{represent covering parameters} \]
Real-world interpretation: Matchings model conflict-free assignments such as student-project pairing, job allocation, team formation, and resource distribution.

Theorem and Proof Cards

These results establish the basic logic of matchings and coverings.

Theorem 1: Every Maximum Matching is Maximal

If \(M\) is a maximum matching of \(G\), then \(M\) is maximal.

Proof. Suppose \(M\) is maximum but not maximal. Then there exists an edge \(e\in E(G)-M\) such that \(M\cup\{e\}\) is also a matching. This new matching has cardinality \(|M|+1\), contradicting the assumption that \(M\) has maximum possible size. Hence every maximum matching is maximal.

Theorem 2: A Maximal Matching Need Not Be Maximum

A matching may be impossible to enlarge locally while still failing to have largest possible size.

Example Proof. In a path \(P_4\), choosing the middle edge gives a maximal matching of size \(1\), since both remaining edges share endpoints with it. However, the two end edges form a matching of size \(2\). Therefore maximal does not necessarily mean maximum.

Theorem 3: Saturation Count

If \(M\) is a matching, then exactly \(2|M|\) vertices are saturated by \(M\).

\[ |\{v\in V(G):v\text{ is saturated by }M\}|=2|M| \]
Proof. Each edge in a matching has two endpoints. Since distinct matching edges share no endpoint, no saturated vertex is counted twice. Therefore the number of saturated vertices is twice the number of matching edges.

Theorem 4: Matching Number Bound

For a graph of order \(n\), the matching number satisfies \[ \alpha'(G)\leq \left\lfloor\frac{n}{2}\right\rfloor. \]

Proof. A matching with \(k\) edges saturates \(2k\) distinct vertices. Since the graph has only \(n\) vertices, \(2k\leq n\). Hence \(k\leq \frac{n}{2}\), and therefore \(k\leq \left\lfloor\frac{n}{2}\right\rfloor\).

Remark: Matching and Covering Philosophy

Matchings select mutually non-conflicting edges. Coverings select graph elements that touch or dominate required structures. Together, they form a central dual viewpoint in graph theory and combinatorial optimization.

\[ \text{Matching: choose disjoint pairs}\qquad \text{Covering: touch required objects} \]

Hands-on Activities

Four activity cards for classroom, lab, and LMS submission.

Activity 1: Pairing Without Conflict

Objective: Understand independent edges.

Procedure: Draw a graph and select edges with no common endpoints.

Expected Outcome: Produce a valid matching.

Python Extension: Validate using endpoint checks.

Activity 2: Maximal vs Maximum Debate

Objective: Distinguish local non-extendability from global optimality.

Procedure: Use \(P_4\) and compare a middle-edge matching with a two-edge matching.

Expected Outcome: Explain why maximal need not be maximum.

Python Extension: Compare matching sizes.

Activity 3: Assignment Network

Objective: Model student-task allocation as matching.

Procedure: Create bipartite-like edges between students and tasks.

Expected Outcome: Find maximum number of conflict-free assignments.

Python Extension: Use nx.max_weight_matching(G).

Activity 4: Saturation Audit

Objective: Identify saturated and unsaturated vertices.

Procedure: Choose a matching and mark all incident vertices.

Expected Outcome: Verify that \(2|M|\) vertices are saturated.

Python Extension: Print saturated vertex set.
